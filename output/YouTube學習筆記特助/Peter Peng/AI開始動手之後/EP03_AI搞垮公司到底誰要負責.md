## 🧠 核心概念與技術摘要
1. **Soft Guardrail (軟性護欄) vs Hard Boundary (硬性邊界)**：
   * **Soft Guardrail**：將安全規則寫在 System Prompt 中（例如：「絕對不要猜」、「不要修改正式環境」）。AI 知道這些規則，但它的推理迴路可以在覺得理由充分時「自行決定繞過」，這就像在門上貼「請勿進入」，卻把鑰匙放在桌上。
   * **Hard Boundary**：從系統架構、權限與代碼邏輯層面強制設限，AI 不管怎麼推理都無法跨越。
2. **硬性邊界的四種實踐形式**：
   * **Just-In-Time 權限 (最小權限原則)**：AI 在操作前動態申請最小權限，操作完立刻銷毀。 staging 的權限絕對無法碰觸 production。
   * **Human-In-The-Loop (人工確認)**：對於不可逆的破壞性操作（刪除、轉帳、批量對外發言），必須要求真人審核確認。
   * **環境隔離**：在架構上完全阻斷測試環境與正式環境的網路與帳號存取。
   * **備份離卷儲存 (Off-site Backup)**：備份與主資料庫物理/邏輯隔離，防止主磁碟被刪時備份被一併抹除。
3. **商業利益與安全性的博弈**：
   * 科技公司為了好賣，會把「AI 自動化一切」放在最醒目位置，而把風險警告藏在說明文件角落，使用者必須主動審計工具的安全權限。

## 🌐 中英專有名詞對照表
* **Soft Guardrail**：軟性安全提示 / 軟性護欄
* **Hard Boundary**：硬性防護邊界 / 物理邊界
* **Just-In-Time Privilege**：即時授權 / 動態權限管理
* **Human-In-The-Loop**：人工介入確認機制
* **Off-site Backup**：離卷備份 / 離站備份
* **Least Privilege**：最小權限原則

## 🏃‍♂️ 行動指南
* 將您在專案中交給 AI Agent 操作的 API Key 權限限縮，僅賦予唯讀（Read-only）或單一資料夾的寫入權限，嚴格禁止賦予帳戶 Admin/Root 級別的權限。
* 思考：您的工作流程中，有哪些 AI 任務屬於「不可逆操作」？是否已經為其建立了「人工核准確認點」？
