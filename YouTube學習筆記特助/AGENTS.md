# YouTube學習筆記特助（由 Gem 升級而來）

這是一個將指定 YouTube 教學影片深度解析、精煉並產出結構化筆記的工作流。
**固定的是流程＋規格（凍在 Skill / 本檔 / scripts），浮動的是材料（每次丟 `input/url.txt`）。**

## 固定偏好（核心守則與未來方針）
- **語言偏好**：回應一律用繁體中文（Taiwan）。
- **筆記規格**：產出筆記必須包含影片資訊、核心概念摘要、附時間戳記的段落大綱、中英術語對照、行動指南。
- **生圖規範**：
  * **預設不生圖**：若使用者未明確要求，不主動進行生圖。
  * **生圖訪談機制**：若使用者要求生圖，必須先與使用者進行對話問答，確認其對當下主題插圖的風格、色彩、比例等具體需求後，才執行生圖。
- **Obsidian 同步**：
  * 所有產出的 Markdown 學習筆記，除了存在專案的 `output/` 目錄外，必須同步寫入 Obsidian 本地儲存庫的 **`YouTube學習筆記/`** 資料夾中。
- **批次與播放清單分組**：
  * 支援在 `input/url.txt` 中填入多個 YouTube 網址（一行一個）。
  * 若偵測到影片屬於同一個「YouTube 播放清單（playlist）」，必須自動建立以該「播放清單標題」命名的子資料夾，將對應的筆記與圖片歸類存放於該子目錄下。

## 怎麼用
1. 將一個或多個 YouTube 影片連結寫入 `input/url.txt`（一行一個）。
2. 執行對應的 `/summarize` 工作流。
3. 產出的 `.md` 筆記會自動儲存於 `output/` 及 Obsidian 中的 `YouTube學習筆記/` 資料夾下。
